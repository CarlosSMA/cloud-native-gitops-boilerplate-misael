# EX-14: Drift Detection and Security

**Arc:** Operate
**Persona:** Security / Platform Engineer
**Time:** Ongoing (continuous)
**Validation Status:** Implicit (ArgoCD reconciliation is always active)

## Product Outcome

ArgoCD continuously reconciles live state against Git. Manual changes via kubectl or AWS Console are detected and reverted. Security posture weakening (security group changes, RBAC modifications) is auto-corrected. Every deployed resource has an audit trail linking to a Git commit.

## Security Posture (Automatic)

These are enforced by default with no user configuration:

### Container Hardening

| Control | Implementation |
|---------|---------------|
| Non-root execution | UID 1000 (runAsUser) |
| Read-only root filesystem | `readOnlyRootFilesystem: true` |
| All capabilities dropped | `drop: ["ALL"]` |
| No privilege escalation | `allowPrivilegeEscalation: false` |
| 32-char random admin password | Auto-generated at deploy |

**Test criteria:**
- [ ] Pod runs as UID 1000 (not root)
- [ ] Writing to root filesystem fails
- [ ] No Linux capabilities available in container
- [ ] Admin password is random (not default)

### Encryption at Rest

| Layer | Mechanism |
|-------|-----------|
| Kubernetes Secrets | KMS envelope encryption |
| EBS volumes | GP3 encrypted (default) |
| S3 buckets | SSE-KMS with bucket key |
| RDS | Encrypted storage |

### Network Isolation

| Control | Implementation |
|---------|---------------|
| Worker nodes | Private subnets only |
| Outbound traffic | Via NAT gateway |
| Security groups | Scoped to VPC CIDR |
| NLB | Internet-facing (explicit opt-in) |
| RDS (port 5432) | VPC CIDR only |
| OpenSearch (port 443) | VPC CIDR only |

### Workload Identity (IRSA)

Every component gets a dedicated IAM role OIDC-bound to its specific service account:

| Component | Role Scope |
|-----------|-----------|
| Liferay pods | S3 document library access |
| EBS CSI driver | EBS volume management |
| S3 CSI driver | S3 mount access |
| AWS LB controller | NLB management |
| External Secrets Operator | Secrets Manager read |
| Crossplane providers (x6) | Per-resource-type AWS access |

**Test criteria:**
- [ ] No static AWS credentials in any pod
- [ ] Each service account has exactly one IAM role
- [ ] Compromising one component does not grant cross-component access

### Credentials Management

| Control | Implementation |
|---------|---------------|
| No secrets in Git | All via ESO from Secrets Manager |
| No secrets in env vars | Mounted as Kubernetes Secrets |
| No secrets in Terraform state | Generated by Crossplane, not Terraform |
| Credential sync | ESO refreshes hourly |
| CI credentials | Append-only (explicit deny on delete) |

## Drift Detection

**How it works:** ArgoCD polls the GitOps repo every 3 minutes. Any divergence between live state and Git-declared state is flagged and auto-corrected.

**Test criteria:**
- [ ] `kubectl edit` a resource -> ArgoCD reverts within sync interval
- [ ] Manual security group change in AWS Console -> Crossplane reverts
- [ ] ArgoCD shows drift event with before/after diff
- [ ] Drift events are logged

### What Gets Protected

| Resource Type | Drift Source | Auto-Corrected |
|---------------|-------------|----------------|
| StatefulSet (replicas, resources) | kubectl edit | Yes (ArgoCD) |
| Service, HTTPRoute | kubectl edit | Yes (ArgoCD) |
| Security Groups | AWS Console | Yes (Crossplane) |
| RDS configuration | AWS Console | Yes (Crossplane) |
| ConfigMaps, Secrets | kubectl edit | Yes (ArgoCD) |

## ArgoCD AppProject Boundaries

Two AppProjects enforce separation:

| AppProject | Scope | Source Repos |
|------------|-------|-------------|
| Infrastructure | Crossplane resources, providers | Infrastructure charts |
| Application | Liferay workloads, services | Application chart + GitOps repo |

**Test criteria:**
- [ ] Application AppProject cannot create infrastructure resources
- [ ] Infrastructure AppProject cannot modify application workloads
- [ ] Source repo allowlist enforced

## Audit Trail

Every change traces back to:
1. Git commit (author, timestamp, diff)
2. ArgoCD sync event (who triggered, what changed)
3. CloudWatch logs (90-day retention)
